# План разработки — Lite (бесплатная веб-страница)
## Box Breathing — зашёл, нажал Старт, дышишь

> Автономная версия без сервера, Telegram и базы данных.  
> Полная версия с группами и ботом: [PLAN.md](./PLAN.md) · Требования: [TZ.md](./TZ.md)

---

## 1. Суть проекта

Одна **веб-страница**, которая работает полностью **в браузере пользователя**.

Пользователь:
1. Открывает ссылку
2. Выбирает параметры (или оставляет по умолчанию)
3. Нажимает **Старт**
4. Видит анимированный квадрат и таймер дыхания

**Сервер не нужен.** Хостинг — бесплатный (статические файлы).

---

## 2. Что входит / не входит

### Входит в Lite MVP

| Функция | Описание |
|---------|----------|
| Фиксированный квадрат | 4 фазы с настраиваемой длительностью (например 4-4-4-4) |
| Прогрессивный квадрат | Рост от начального до конечного значения, шаг, число проходов |
| Ограничение | По времени (10/20/30/60 мин) **или** по количеству проходов |
| Обратный отсчёт | 3 секунды перед стартом (фиксированно в MVP) |
| Визуальный квадрат | Подсветка фазы, индикатор по периметру |
| Управление | Старт, Пауза, Продолжить, Стоп |
| Адаптив | Телефон и ПК |
| Сохранение настроек | `localStorage` — последние параметры при следующем визите |

### Не входит (только в [PLAN.md](./PLAN.md))

| Функция | Почему |
|---------|--------|
| Групповая синхронизация | Нужен сервер и WebSocket |
| Telegram-бот | Нужен backend и токен |
| Статистика участников | Нет общей базы |
| Комнаты, история на сервере | Нет backend |
| Режимы: Пирамида, Волна | Отложены даже в Lite v2 |
| Звук, фоны | Отложено (можно добавить позже, всё ещё без сервера) |

### Опционально в Lite (не блокирует MVP)

| Функция | Хранение |
|---------|----------|
| Локальная история сессий | `localStorage` на устройстве пользователя |
| Экспорт настроек в ссылку | Query-параметры URL (`?mode=fixed&in=4&...`) |

## 4. Стек и почему он простой

| Слой | Рекомендация | Альтернатива |
|------|--------------|--------------|
| Разметка | HTML5 | — |
| Стили | CSS3 (без фреймворка) | Tailwind (если удобнее) |
| Логика | Vanilla TypeScript | React + Vite (если планируем потом миграцию в Web App) |
| Сборка | Vite | Можно и без сборки — один `index.html` |
| Тесты | Vitest (модуль дыхания) | Опционально |
| Деплой | GitHub Pages | Cloudflare Pages, Netlify |

**Рекомендация для Lite:** `Vite + TypeScript + vanilla` — быстро, легко, потом модуль `breathing` можно перенести в полную версию.

### Экран 2 — Практика

```
┌────────────────────────────────┐
│                                │
│         ┌──────────┐           │
│         │   ВДОХ   │ ← активно │
│    ЗАДЕР│          │ЗАДЕР      │

│   Прогресс: 35%                │

│                                │
│   [ Пауза ]  [ Стоп ]          │
└────────────────────────────────┘
```

### Поток

1. Загрузка страницы → подтянуть настройки из `localStorage`
2. Пользователь жмёт **Старт**
3. Обратный отсчёт 3-2-1
4. Движок запускает фазы по расписанию
5. UI обновляется каждый кадр / каждую секунду
6. По окончании — экран «Практика завершена» + кнопка «Начать снова»
7. Опционально: запись в локальную историю

## 7. Модуль дыхания (ядро)

Тот же алгоритм, что в полной версии ([PLAN.md](./PLAN.md), Этап 1), но целиком на клиенте.

### Фазы

```
INHALE → HOLD_IN → EXHALE → HOLD_OUT → (следующий цикл)
```

### Режим «Фиксированный»

Все циклы с одинаковой длительностью фаз.

### Режим «Прогрессивный»

После каждого полного цикла все 4 значения увеличиваются на `step` до `end_value`.  
Достигли `end_value` → +1 проход → сброс к `start_value`.

Пример (3 прохода, 4→20, шаг 1):

```
Проход 1: 4-4-4-4, 5-5-5-5, … 20-20-20-20
Проход 2: 4-4-4-4, 5-5-5-5, … 20-20-20-20
Проход 3: 4-4-4-4, 5-5-5-5, … 20-20-20-20
```

### Ограничение по времени

Движок заранее считает расписание циклов и обрезает, когда суммарное время ≥ лимита.

### API модуля (черновик)

```typescript
interface BreathingConfig {
  mode: 'fixed' | 'progressive';
  inhale: number;
  holdIn: number;
  exhale: number;
  holdOut: number;
  // progressive only:
  startValue?: number;
  endValue?: number;
  step?: number;
  passes?: number;
  // limit:
  limitType: 'time' | 'passes';
  durationMinutes?: number;
}

interface PhaseState {
  phase: 'inhale' | 'hold_in' | 'exhale' | 'hold_out';
  secondsLeft: number;
  currentLevel: number;   // текущая длительность стороны
  pass: number;
  totalPasses: number;
  progressPercent: number;
  totalSecondsLeft: number;
}

// Создать расписание → запустить → колбэк onTick(state)
```

## 9. Хостинг и стоимость

| Статья | Стоимость |
|--------|-----------|
| GitHub Pages / Cloudflare Pages | **0 ₽** |
| Домен (опционально) | ~200–500 ₽/год |
| VPS | **Не нужен** |
| База данных | **Не нужна** |
| Telegram Bot Token | **Не нужен** |

### Деплой на GitHub Pages (кратко)

1. Репозиторий на GitHub
2. `npm run build` → папка `dist/`
3. GitHub Actions или ветка `gh-pages` — отдача статики
4. URL: `https://<user>.github.io/<repo>/`

### Деплой на Cloudflare Pages

1. Подключить репозиторий
2. Build command: `npm run build`
3. Output directory: `dist`
4. Бесплатный SSL и CDN

## 10. Ограничения Lite (честно)

| Ограничение | Пояснение |
|-------------|-----------|
| Нет синхронизации с другими | Каждый дышит в своём темпе на своём устройстве |
| История только локальная | Сменил телефон / очистил браузер — история пропала |
| Таймер зависит от вкладки | В фоне браузер может замедлять таймеры (решается частично через Web Audio / timestamp) |
| Нет модерации и аккаунтов | Открытая страница для всех |

**Улучшение точности таймера:** считать время через `Date.now()` и разницу меток, а не слепой `setInterval` — заложить в Этап 1.

## 11. Связь с полной версией

Lite не выбрасывается при переходе к [PLAN.md](./PLAN.md):

| Компонент Lite | Куда переедет |
|----------------|---------------|
| `breathing/engine.ts` | `backend/app/services/breathing/` + `frontend/` |
| UI квадрата | Telegram Web App |
| Стили и UX | Основа Web App |

**Стратегия:** сначала Lite → проверить UX и логику дыхания → потом обернуть в Telegram + сервер для групп.

```
PLAN-LITE.md  ──►  рабочая страница  ──►  PLAN.md (бот + группы)
     │                                        │
     └──── модуль breathing переиспользуется ─┘
```
