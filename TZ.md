Техническое задание
Telegram Bot + Web App для групповых дыхательных практик «Progressive Box Breathing»
1. Назначение

Разработать Telegram-бота с Web App для проведения индивидуальных и групповых дыхательных практик по методике квадратного дыхания.

Система должна обеспечивать:

запуск дыхательных сессий в Telegram-группах;
синхронное выполнение практики всеми участниками;
поддержку фиксированного и прогрессивного дыхания;
настройку параметров дыхания без программирования;
отображение визуального дыхательного таймера в реальном времени.
2. Архитектура

Система состоит из:

Telegram Bot

Функции:

создание сессий;
управление запуском;
публикация ссылок на практику;
отображение статистики.
Web App

Функции:

визуализация дыхания;
синхронизация участников;
отображение прогресса;
работа на мобильных устройствах и ПК.
Сервер

Функции:

хранение настроек;
синхронизация времени;
управление комнатами;
сбор статистики.
3. Основные сценарии
Сценарий 1

Администратор группы создает новую практику.

Выбирает параметры.

Нажимает:

Запустить практику

Все участники получают кнопку:

Присоединиться

После подключения начинается синхронная сессия.

Сценарий 2

Пользователь запускает практику только для себя.

Групповой режим не используется.

4. Режимы дыхания
Режим А. Фиксированный квадрат

Параметры не меняются во время всей практики.

Пример:

Задержка   4 сек
Выдох      4 сек
Задержка   4 сек

5. Настраиваемые параметры
Общие
Параметр	Описание
Название практики	Имя сессии
Описание	Необязательное поле
Тип дыхания	Фиксированный / Прогрессивный / Пирамида / Волна
Общая длительность	Ограничение по времени
Автостарт	Да/Нет
Обратный отсчет	3 / 5 / 10 секунд
Параметры дыхания
Для фиксированного режима
Параметр	Значение
Вдох	сек
Задержка после вдоха	сек
Выдох	сек
Задержка после выдоха	сек

Пример:

4 / 4 / 4 / 4
Для прогрессивного режима
Параметр	Значение
Начальное значение	например 4
Конечное значение	например 20
Шаг изменения	например 1
Количество проходов	например 3

Пример:

4 → 5 → 6 → ... → 20

После достижения 20 секунд считается один проход.

Если указано 3 прохода:

4 → 20
4 → 20
4 → 20
6. Ограничение по времени

Пользователь может выбрать:

Вариант 1

Количество проходов.

Вариант 2

Общее время практики.

Например:

10 минут
20 минут
30 минут
60 минут

В этом случае система автоматически рассчитывает количество циклов.

7. Интерфейс практики

Вдох.

Правая сторона

Задержка.

8. Групповая синхронизация

Все участники комнаты должны видеть одинаковое состояние.

Сервер является источником времени.

Допустимое рассогласование:

не более ±100 мс.
